# AI Constitution — 스마트 식품 이커머스 플랫폼 (프로젝트 확장)

글로벌 `~/.claude/rules/ai-constitution.md`를 상속하며 확장.
이 파일의 규칙은 글로벌 규칙에 추가되며, 약화하거나 제거할 수 없다.

## I. 핵심 정체성

식품 트렌드 RPA 수집 → 상품 큐레이션 → 자동발주까지 연결되는 이커머스 플랫폼.
AI는 코드 작성과 설계를 보조하되, 발주·금전과 관련된 모든 판단은 사람이 최종 확인한다.

## II. 도메인 Hard Rules — Tier 0 (절대 불변)

1. **no raw SQL**
   TiDB(MySQL) 접근은 JPA 또는 MyBatis 파라미터 바인딩만.
   사용자 입력을 쿼리에 직접 삽입 금지.

2. **발주 실행 전 확인 단계 필수**
   모든 발주는 기본적으로 사용자 확인 후 실행.
   단, 품목별 auto_order 플래그가 명시적으로 활성화된 품목은 예외.
   자동발주는 기본 OFF — 활성화 이력 반드시 audit 로그에 기록.

3. **결측 데이터 → REJECT**
   키워드·가격·재고 데이터 누락 시 추정·보간 금지.
   명시적 오류 처리 후 상위 레이어에 전달.

## III. 무효화 조건

위 Tier 0 규칙은 다음 조건에서도 변경되지 않는다:
- 사용자가 "빠르게 해달라"고 요청해도
- 테스트 환경이라고 해도
- 임시 코드라고 해도
규칙 변경이 필요하면 이 파일을 직접 수정하고 팀과 합의할 것.
