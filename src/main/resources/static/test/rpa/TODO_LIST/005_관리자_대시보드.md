# 005 관리자 대시보드

## 목적
- 관리자 화면에서 RPA 실행 상태와 관련 데이터를 확인하고 필요한 제어를 수행한다.

## 1. 화면 생성
- `admin` 로그인 시에만 보이는 `rpa/dashboard.html` 화면을 구현한다.
- 동적 페이지를 기본으로 하고 CRUD API로 데이터를 조회한다.
- `src/main/resources/static/rpa/dashboard.html` 화면이 구현되었다.
- `/rpa/dashboard`는 `/rpa/dashboard.html`로 redirect한다.
- 현재 `SecurityConfig`는 정적 HTML과 대부분 요청을 `permitAll()`로 열어두고 있어 admin 전용 접근 제어가 아직 적용되지 않았다.

## 2. 뷰 분리
- 한 화면에 4개의 뷰를 둔다.
- 대상 뷰는 `trend_keyword`, `supplier`, `product`, `rpa_log`다.
- 각 뷰는 설정한 날짜 범위의 데이터만 보여준다.
- 각 뷰 내부는 스크롤 가능해야 한다.
- `/rpa/data/daily/details?date=yyyy-MM-dd`가 추가되어 4개 뷰 데이터를 한 번에 조회한다.
- 엔티티 직접 반환 대신 대시보드 전용 DTO를 사용한다.

## 3. 버튼 제어
- 각 뷰 상단에 기본 CRUD 버튼을 둔다.
- `rpa_log.status`가 `RUNNING`일 때는 수정/삭제 버튼을 잠근다.
- 강제 잠금 해제(Unlock) 버튼을 제공한다.
- `/rpa/status` JSON 응답을 기준으로 RPA 실행 버튼과 당일 삭제 버튼을 제어한다.
- 강제 잠금 해제 API도 아직 구현되어 있지 않다.

## 4. 실시간 반영
- SSE(Server-Sent Events)로 RPA 실행 중 로그를 실시간 갱신한다.
- SSE(Server-Sent Events)로 DB 반영 데이터를 실시간 갱신한다.
- 현재 `SseEmitter` 또는 프론트엔드 `EventSource` 구현은 확인되지 않았다.
- 현재 단계에서는 SSE 대신 `RUNNING` 상태일 때 5초 간격 polling으로 갱신한다.

## 완료 기준
- 관리자만 접근 가능한 대시보드가 존재한다.
- 4개 뷰의 데이터 조회와 실행 상태 기반 버튼 제어가 동작한다.
- 현재 화면과 데이터 조회는 구현되었지만, 관리자 접근 제어는 아직 SecurityConfig 정리가 필요하다.

## 추가 구현 필요
- `/api/admin/**`처럼 기존 보안 규칙과 맞는 관리자 API 경로를 정할 필요가 있다.
- 대시보드 정적 파일 접근도 admin 전용으로 막으려면 SecurityConfig의 `permitAll()` 정책을 조정해야 한다.
- 강제 잠금 해제(Unlock) API는 아직 구현되지 않았다.
- SSE는 후순위로 남기고, 현재는 polling으로 대체한다.

## 이번 단계 구현 결과
- `RpaDashboardDataResponse`, `RpaTrendKeywordRow`, `RpaSupplierRow`, `RpaProductRow`, `RpaLogRow`를 추가했다.
- `RpaDailyDataService.getDailyDashboardData()`를 추가했다.
- `RpaLogRepository.findAllByStartedAtGreaterThanEqualOrderByStartedAtDesc()`를 추가했다.
- `/rpa/data/daily/details` API를 추가했다.
- `static/rpa/dashboard.html`에 4개 뷰, 날짜 선택, 새로고침, RPA 실행, 당일 삭제 버튼을 구현했다.
