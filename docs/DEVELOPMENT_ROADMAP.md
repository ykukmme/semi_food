# 스마트 식품 이커머스 플랫폼 — 개발 로드맵

## Phase 1: Foundation (목표: 기본 구조 동작)

- [ ] 1-1. 프로젝트 구조 셋업 (backend / rpa / frontend 분리)
- [ ] 1-2. TiDB Cloud 연결 및 DB 스키마 설계 (상품, 트렌드 키워드, 주문, 발주 테이블)
- [ ] 1-3. Spring Boot 기본 API 서버 구동 확인
- [ ] 1-4. Python RPA 환경 셋업 (Playwright 또는 Selenium)
- [ ] 1-5. 기본 테스트 구조 구성 (JUnit5 + pytest)

## Phase 2: 트렌드 수집 및 분석

- [ ] 2-1. 네이버 쇼핑 RPA 수집기 구현 (키워드 + 카테고리)
- [ ] 2-2. 수집 데이터 정제 및 TiDB 저장 파이프라인
- [ ] 2-3. 트렌드 키워드 분석 로직 (빈도, 상승세 계산)
- [ ] 2-4. 수집 스케줄러 구현 (cron 기반 주기적 실행)
- [ ] 2-5. 수집 결과 조회 REST API (Spring Boot)

## Phase 3: 상품 큐레이션 자동화

- [ ] 3-1. 트렌드 키워드 기반 상품 매핑 로직
- [ ] 3-2. 큐레이션 결과 저장 및 조회 API
- [ ] 3-3. 프론트엔드 큐레이션 대시보드 (트렌드 + 추천 상품 표시)
- [ ] 3-4. 상품별 자동발주 플래그 설정 UI

## Phase 4: 주문 및 발주 시스템

- [ ] 4-1. 주문 접수 API 및 주문 상태 관리
- [ ] 4-2. 발주 확인 단계 구현 (기본: 수동 확인)
- [ ] 4-3. 품목별 자동발주 플래그 연동 (활성화 시 확인 단계 건너뜀)
- [ ] 4-4. 자동발주 이력 로깅 (누가, 언제, 어떤 품목을 활성화/발주했는지)
- [ ] 4-5. 발주 현황 대시보드
