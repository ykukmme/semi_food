# 001 RPA 파싱 준비

## 목적
- 당일 트렌드 키워드를 저장하고, 후속 공급자/상품 파싱에 필요한 기준 데이터를 준비한다.

## 1. 트렌드 키워드 저장 API 연결
- 트렌드 키워드 저장 API를 호출한다.
- 예시 URL은 `http://localhost:8080/api/TrendKeywords/saveWithSequentialId`다.
- 현재 `ParserController.saveWithSequentialId()`에 `/api/TrendKeywords/saveWithSequentialId` 엔드포인트가 구현되어 있다.
- 현재 메서드는 별도 `size` 파라미터를 받지 않고, `TrendKeywordService.getNaverKeywords()` 결과 전체를 저장 대상으로 사용한다.

## 2. 저장 결과 식별
- 저장된 트렌드 키워드에서 `keywordId`, `rankId`, `syncDate`를 얻을 수 있어야 한다.
- 최근 저장된 20건을 기준으로 다음 단계가 진행되어야 한다.
- 현재 `TrendKeyword` 엔티티에는 `id`, `rankingId`, `syncDate`, `collectedAt`, `rank`가 존재한다.
- 현재 `V2__create_trend_keyword_table.sql` 기준으로 `ranking_id BIGINT`, `sync_date DATETIME` 컬럼이 존재한다.
- 현재 `TrendKeywordRepository`에는 `findMaxId()`, `existsByCollectedAtAndKeyword()`, `findFirstByOrderByIdDesc()`가 구현되어 있다.
- `findRecentRpaReadyKeywords(LocalDateTime start, Pageable pageable)`가 추가되어 당일 기준 후속 파싱 가능 키워드를 조회할 수 있다.

## 3. 당일 데이터 기준 정렬
- 당일 파싱한 `TrendKeyword` 목록만 대상으로 삼는다.
- 실행 시점의 날짜와 트렌드 순위를 함께 기록한다.
- 제품과 공급자 데이터는 실행 시점 기준의 최신 데이터를 사용한다.
- `TrendKeywordService.getTodayRpaParseTargets(int requestedSize)`에서 당일 0시 이후 데이터를 기준으로 RPA 파싱 타깃을 준비한다.
- 현재 최신성 판단은 `TrendKeywordService.saveWithSequentialId()`에서 DB 최신 레코드의 `collectedAt` 날짜와 비교하는 방식이다.

## 완료 기준
- 당일 기준 `TrendKeyword` 20건을 후속 작업 입력값으로 사용할 수 있다.
- `keywordId`, `rankId`, `syncDate` 조합이 동적으로 준비된다.
- `/api/TrendKeywords/rpaTargets?size=20`로 후속 파싱 입력값을 확인할 수 있다.

## 추가 구현 필요
- 전체 RPA 실행 시퀀스에서는 `/api/TrendKeywords/rpaTargets` 결과를 사용해 공급자/상품 파싱을 반복 호출해야 한다.
- 같은 날짜 데이터가 이미 있는 경우 현재 서비스가 안내용 더미 `TrendKeyword(id=0)`를 반환하므로, RPA 시퀀스에서는 이 값을 실제 입력값으로 사용하지 않도록 방어해야 한다.

## 이번 단계 구현 결과
- `RpaTrendKeywordParseTarget` record를 추가해 `keywordId`, `rankId`, `syncDate`, `trendRank`, `collectedAt`를 명시적으로 반환한다.
- `syncDate`는 `LocalDateTime`에서 네이버 API 호출용 `yyyyMMdd` 문자열로 변환한다.
- 당일 기준 조회 시 `id > 0`, `rankingId IS NOT NULL`, `syncDate IS NOT NULL` 조건으로 안내용 더미 데이터와 불완전 데이터를 제외한다.
