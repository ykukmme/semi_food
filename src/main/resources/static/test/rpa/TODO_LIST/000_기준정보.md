# 000 기준 정보

## 목적
- 이 문서는 `semi_food` 프로젝트의 RPA 구현 작업을 시작하기 전에 반드시 확인해야 하는 기준 정보다.

## 기준 스키마
- RPA 상태의 기준은 `src/main/resources/db/migration/V6__create_rpa_log_and_audit_tables.sql`과 `src/main/resources/db/migration/V7__alter_status_columns_to_varchar.sql`이다.
- `rpa_log.status` 값은 `RUNNING`, `COMPLETED`, `FAILED`를 사용한다.
- `rpa_log`의 주요 컬럼은 `started_at`, `ended_at`, `keyword_count`, `product_count`, `message`다.
- V6에서는 `rpa_log.status`가 `ENUM`으로 생성되고, V7에서 `VARCHAR(10)`으로 변경된다.
- `trend_keyword.sync_date`는 `DATETIME`이며, `TrendKeyword.syncDate`의 `LocalDateTime` 매핑과 일치한다.
- `product.sync_date`는 `DATETIME`이며, `Product.syncDate`의 `LocalDateTime` 매핑과 일치한다.

## 로그 기준
- RPA 로그 파일 저장 위치는 `src/main/resources/static/test/rpa/log`다.
- 로그 파일명 규칙은 `rpa_parsing_yymmdd_time.log`다.

## 구현 기준
- 예시 URL과 예시 파라미터 값은 실제 구현 기준이 아니라 샘플이다.
- 실제 코드 작성 시에는 현재 엔티티와 마이그레이션 스키마를 우선한다.

## 현재 구현 상태
- `RpaLog`, `RpaStatus`, `RpaLogRepository`, `RpaRestController` 클래스가 존재한다.
- `RpaStatus`는 `RUNNING`, `COMPLETED`, `FAILED`만 정의되어 있다.
- `RpaLog.complete()`와 `RpaLog.fail()` 메서드로 완료/실패 상태 전이가 일부 구현되어 있다.
- `RpaRestController`의 `/rpa/dashboard`, `/rpa/status`, `/rpa/run`은 현재 placeholder 응답만 반환한다.
- `RpaLog.id`는 TiDB 이슈로 `@GeneratedValue`가 주석 처리되어 있어 수동 ID 부여 정책과 충돌하지 않게 구현해야 한다.

## 보충 필요
- `TrendKeyword.syncDate`, `Product.syncDate`는 현재 마이그레이션 기준으로 `DATETIME` 컬럼과 매핑된다.
- 개발용 Flyway 설정에서 `validateOnMigrate(false)`, `outOfOrder(true)`가 사용 중이므로 배포 전 마이그레이션 검증 정책을 정리해야 한다.
- `supplier`에는 별도 `sync_date`가 없으므로 공급자 데이터의 실행 기준 날짜는 `created_at` 또는 연결된 상품/키워드 기준으로 해석해야 한다.
